## What is Afiax?

Afiax Connected Healthcare is a **pan-African digital health platform** being built on a Medplum-derived,
FHIR-native clinical core. Kenya is the first country implementation, but the platform itself is broader than the Kenya
pack.

Afiax combines:

- **Clinical core**: canonical FHIR resources, access control, subscriptions, Bots, operations, and shared domain services
- **Digital service layers**: provider workflows, patient engagement, telemedicine, analytics, and partner APIs
- **Country packs**: national terminology, registries, payer rails, claims, and exchange connectors
- **Tenant overlays**: customer-specific configuration, workflow toggles, and deployment choices
- **Delivery models**: SaaS, managed PaaS, and infrastructure patterns aligned to local sovereignty and scale needs

## Platform scope

- Cloud-based electronic health record and clinical operations foundation
- Interoperability and national exchange services built on HL7/FHIR
- Telemedicine and remote-care enablement
- Analytics and AI-driven decision support
- Developer and partner platform capabilities via APIs, Bots, and country-pack contracts
- Shared SaaS, dedicated runtime, and managed deployment options depending on market and customer needs

## How the docs are organized

1. Start with the [architecture overview](/docs/architecture) for the broad Afiax platform model.
2. Use the [enterprise platform](/docs/architecture/enterprise-platform) and [canonical FHIR model](/docs/architecture/canonical-model) docs to define the shared core.
3. Open [country packs](/docs/country-packs) when you are ready to localize the platform for a market.
4. Use the [Kenya reference pack](/docs/country-packs/kenya) as the first implementation of that model.
5. Use [Bots](/docs/bots), [FHIR operations](/docs/api/fhir/operations), and the [FHIR datastore](/docs/fhir-datastore) to deliver auditable workflows.

## Core principles

- Country-neutral core, country-specific overlays
- FHIR-native source of truth in Medplum resources
- No direct national API calls from UI applications
- Sovereign deployment for identifiable clinical data
- Audit, provenance, and reconciliation before production go-live

## Why Kenya appears in the docs

Kenya is documented because it is the **first reference country pack**, not because it defines the whole Afiax
platform. The shared core, SDK contracts, deployment models, and operating principles are intended to support
additional African markets without rewriting the platform.
